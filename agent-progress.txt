**API 接口列表**:
- POST /api/address/add - 新增收货地址
- PUT /api/address/update - 修改收货地址
- DELETE /api/address/delete/{id} - 删除收货地址
- GET /api/address/{id} - 根据 ID 查询地址
- GET /api/address/list - 查询用户地址列表
- POST /api/address/default/{id} - 设置默认地址

**核心业务逻辑**:
1. 新增地址时，如果设置为默认地址，自动取消其他地址的默认状态
2. 修改地址时，如果设置为默认地址，自动取消其他地址的默认状态（排除当前地址）
3. 删除地址时进行权限验证，确保只能删除自己的地址
4. 查询地址列表时，默认地址排在最前面，按更新时间降序排序
5. 设置默认地址时，先取消当前默认地址，再设置新的默认地址

**技术要点**:
- 使用雪花算法生成唯一 ID（@TableId(type = IdType.ASSIGN_ID)）
- 使用 @Transactional 注解保证事务一致性
- 使用 @Validated 注解进行参数校验
- 使用 LambdaQueryWrapper 进行类型安全的查询构建

## 技术决策记录

### 前端框架选择
- **选择**: uni-app
- **原因**: 
  - 一套代码多端发布 (微信小程序、H5、App)
  - 基于 Vue 3，学习成本低
  - 丰富的组件生态 (uView UI)

### 后端框架选择
- **选择**: Spring Boot 2.7+
- **原因**:
  - 成熟的生态系统
  - 快速开发
  - 易于集成 MyBatis-Plus、Redis 等

### 数据库选择
- **选择**: MySQL 8.0+
- **原因**:
  - 团队熟悉
  - 文档丰富
  - 性能满足需求
